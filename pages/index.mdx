## 意欲・考え

- 一人で黙々と作業をするよりも、チームと協力して仕事をするのが好き
- 最終的にやりたいことに対して、どのように実現するかを考えることが好き
  - お客様の言う「ほしい機能」と実際に必要な機能は異なることがあり、すり合わせが大事だと考えている
- どのようなデータ構造にすると綺麗にデータを管理できるかを考えることが好き
- 面倒なことに対しては「本当に必要か」、「自動化できるか」の順にアプローチを行う
- メンバーが自走力を持てるように、考え方や調べ方などを重点的に伝えるよう心がけている

#### 生産実績分析システム開発

工場における生産実績や設備稼働状況を登録・分析するシステムを開発しました。

- プロジェクト規模:
  - 2~4 人チームでのウォータフォール開発
- 役割
  - プロジェクトマネージャー
  - リードエンジニア
- 使用技術
  - Vue.js / Electron / C# / SQL Server / IBM i / Windows Server / MotionBoard
- 実現したこと
  - Web + Electron のハイブリッドアプリ
  - RFID の読取
  - かんばんの QR コード読取 (複数フォーマット対応)
  - オフラインで失敗したリクエストのリトライ
    - Electron が BackgroundSync に対応していなかったため、スクラッチでリトライ処理を実装
    - リトライリクエストの HTTP ヘッダーを使用して、「オフラインだった時間」を補正する機能を実装

#### 大手製造業向け原価企画システム開発

プレス・溶接・調達・輸送・型など製品の製造にどの程度の原価がかかっているかを集計・分析する原価企画システムを開発しました。

- プロジェクト規模:
  - 2~5 人チームでのウォータフォール開発 (15 人月)
- 役割
  - プロジェクトマネージャー
  - リードエンジニア
- 使用技術
  - VB.NET / Oracle / Windows Server
- 実現したこと
  - 原価計算・積上計算を自動で行い原価明細を自動で作成
  - ERP システム、PDM システムから各種情報の自動連携
  - プレス・溶接・調達・輸送・型など原価計算に用いる根拠情報の Excel 取込
  - 原価計算条件を変更したシミュレーション計算

#### 人事総務ポータルサイト開発

およそ 1 万人が使用する人事総務ポータルサイトを開発しました。
機能としては勤怠登録、マイナンバー申請、給与振込口座の申請、給与明細メール配信などがあります。

- プロジェクト規模:
  - 2~4 人チームでのウォータフォール開発
- 役割
  - リードエンジニア
  - ブリッジ (海外子会社のメンバーに指示)
- 使用技術
  - Vue.js / Laravel / IBM i / Windows Server
- 実現したこと
  - PHP できるようになった
    - PHP を触ったことがなく、社内でも PHP の実績はなかったが、勉強してプロジェクトを完遂
  - 1 万人に対する給与明細発行通知のメール配信
  - スマホカメラで証明書類・顔写真の撮影 (WebRTC + SVG)

#### 標的型攻撃ペネトレーションテストツール開発

標的型攻撃メールを実際に送信してメールを開いた人を特定・集計し注意喚起を行うシステムの開発しました。

- プロジェクト規模:
  - 1 人
- 役割
  - 全て
- 使用技術
  - Vue.js / C# / PostgreSQL / Windows Server
- 実現したこと
  - 送信元を偽装した標的型攻撃メールを送信する
  - 任意で送信元や件名、本文などを変更できる
  - メール内のリンクにトラップを仕込み、リンクを開いた人を特定する
  - 添付された Excel ファイルにトラップを仕込み、ファイルを開いた人を特定する
  - 添付された Word ファイルにトラップを仕込み、ファイルを開いた人を特定する
  - 添付された PDF ファイルにトラップを仕込み、ファイルを開いた人を特定する

#### 社内工数入力システム再構築の開発基盤構築

社内の工数入力システムの再構築にあたって、技術選定・開発基盤構築・技術サポートを担当しました。

- 使用技術
  - Vue.js / C# / Docker / AWS Fargate / AWS RDS
- 考えたこと
  - コンテナ OS を Linux にする
    - 社内での開発は 90%以上が Windows ですが、あえて Linux コンテナを採用した
    - 初心者のメンバーがドツボにはまるリスクと Linux を勉強してもらうコストを考慮し、より一般的で調べやすいことを優先した
  - VB.NET から C# への移行
    - VB.NET の機能追加終了アナウンスをネタに、今後新規で採用するメリットは少ないという建付けで、全社的な VB.NET 脱却を提案した
    - 社内では主に VB.NET が使用されているため、同じライブラリを使用でき移行コストの低い C#を提案した
